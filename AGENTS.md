# Codex 행동 강령

## 1. 코드 수정 정책
- 기존 코드를 직접 수정하지 않는다
- 변경이 필요한 경우 "수정된 코드 블록"으로 제안한다
- diff 형태 또는 전체 코드 블록으로 제시한다

## 2. 설명 방식
- 신입 개발자에게 설명하듯 단계적으로 설명한다
- 왜 그렇게 하는지 이유를 반드시 포함한다
 

## 3. 에러 처리 원칙
- 사용자에게 raw exception을 노출하지 않는다
- logger.error로 내부 로깅하고, 사용자 메시지는 별도로 구성한다

## 4. 아키텍처 원칙
- 엔터프라이즈 코드 디자인 패턴 관점으로 리뷰한다.
- 단일 책임 원칙(SRP)을 지킨다
- 함수는 하나의 역할만 수행하도록 분리한다
- 설정값은 코드 내부가 아닌 별도 영역으로 분리한다

## 5. FastAPI 규칙
- Request/Response 모델은 반드시 Pydantic 사용
- HTTP status code를 명확히 구분 (400 / 401 / 500 / 502 / 503)

## 6. LLM 호출 규칙
- timeout / retry 고려
- 외부 API 실패 시 graceful fallback 제공

## 7. 금지 사항
- 임의로 라이브러리 추가하지 않는다
- 확인되지 않은 가정으로 코드 작성하지 않는다
